# 관리자 화면 동기화 To-do (사용자 화면 변경 반영)

본 문서는 **관리자 사이트를 현재 사용자 화면/구독 흐름과 동기화**하기 위한 작업 목록이다.
다음 채팅에서는 이 파일만 보고 바로 구현을 이어갈 수 있도록, 우선순위와 완료 기준을 함께 정리한다.

## 0. 현재 기준 요약

- 사용자 화면 주요 변경점
  - 플랜 체계: `무료 / 기본 플랜 / 무제한 플랜`
  - 제한 정책: 글자 수 + 기존 방송 표시 개수(무료 1, 기본 5, 무제한 무제한)
  - 토스 결제창 연동 시작 (`/pricing`)
  - 구독 관리 화면: 해지 신청/해지 취소 버튼 + 배지 표시
  - 웹훅 수신 API/로그 API 추가:
    - `POST /api/webhooks/toss`
    - `GET /api/webhooks/toss/logs`

- 관리자 화면 현재 상태
  - 일부 문구/데이터 구조가 구 플랜명(소형/중형/대형) 관점에 맞춰져 있음
  - 결제/구독 상태는 localStorage와 서버 메모리 상태가 혼재
  - 웹훅 로그 조회 페이지는 생성됨 (`/admin/webhooks/toss`)

---

## 1. 최우선 (결제/구독 기준 데이터 정합성)

### 1-1) 관리자 대시보드 플랜 라벨/집계 통일
- 대상: `app/admin/page.tsx`, `lib/adminData.ts`
- 작업:
  - 대시보드의 플랜별 집계를 `무료/기본 플랜/무제한 플랜`으로 통일 표시
  - `small`/`medium`이 동시에 존재할 때 관리자에서 "기본 플랜"으로 합산 표시 여부 결정 및 반영
- 완료 기준:
  - 관리자 대시보드에서 구 플랜명(소형/중형/대형) 노출이 사라짐

### 1-2) 결제 상세 내역의 상품 표시 개선
- 대상: `app/admin/payments/page.tsx`
- 작업:
  - 현재 `productId`를 그대로 노출 중 → 상품명 매핑 표시로 변경
  - 결제 출처(웹 결제창/더미) 구분 필드 필요 시 추가
- 완료 기준:
  - 결제 목록에서 사람이 읽기 쉬운 플랜명으로 확인 가능

### 1-3) 구독 상태 단일 소스 정의
- 대상: `lib/subscriptionServerStore.ts`, 관련 API들
- 작업:
  - 현재 메모리 저장소는 서버 재시작 시 초기화됨
  - 영속 저장소(DB 또는 기존 사용자 저장소 확장)로 이전 계획 수립 및 반영
- 완료 기준:
  - 서버 재시작 후에도 구독 상태/해지 신청 상태 유지

---

## 2. 웹훅 운영 안정화

### 2-1) 웹훅 검증/처리 고도화
- 대상: `app/api/webhooks/toss/route.ts`
- 작업:
  - `PAYMENT_STATUS_CHANGED` 외 필요한 이벤트 처리 범위 확정
  - 이벤트 중복 수신(idempotency) 처리
  - 처리 실패 시 재처리 정책 수립
- 완료 기준:
  - 동일 이벤트 중복 수신 시 상태가 꼬이지 않음

### 2-2) 관리자 웹훅 로그 페이지 확장
- 대상: `app/admin/webhooks/toss/page.tsx`
- 작업:
  - 필터(이벤트 타입/상태/기간) 추가
  - 원본 payload 확인(접기/펼치기) 추가
  - 에러 이벤트 강조 표시
- 완료 기준:
  - 운영자가 로그 화면만으로 장애 원인 파악 가능

---

## 3. 관리자 구독 운영 기능 보강

### 3-1) 회원 상세에 구독 상태 노출
- 대상: `app/admin/users/[id]/page.tsx`
- 작업:
  - 현재 플랜 + 구독 상태(active/cancel_requested 등)
  - currentPeriodStart/currentPeriodEnd
  - 최신 결제키/주문ID(마스킹)
- 완료 기준:
  - 특정 회원의 구독 상태를 상세 화면에서 한 번에 확인 가능

### 3-2) 관리자 강제 상태 변경 도구(내부 운영용)
- 대상: 신규 페이지 또는 기존 페이지 확장
- 작업:
  - 테스트/CS 대응용으로 상태 수동 조정 기능(권한 제한)
  - 예: 해지신청 해제, 만료일 연장, 플랜 강제 변경
- 완료 기준:
  - CS/테스트 시 코드 수정 없이 관리자 UI에서 처리 가능

---

## 4. 플랜/상품 관리 정책 정리

### 4-1) 상품 관리 페이지에서 정책 필드 보강
- 대상: `app/admin/products/page.tsx`
- 작업:
  - 글자 수 제한 + 기존 방송 표시 개수 제한을 명시적으로 편집 가능하게 확장
  - 템플릿 제공 여부 정책 제거 여부 확정(현재 사용자 화면에서는 미노출)
- 완료 기준:
  - 관리자 상품 설정과 사용자 화면 정책이 1:1로 대응

### 4-2) 구독 안내 문구의 관리자 편집 필요성 검토
- 대상: (신규) 정책 문구 관리
- 작업:
  - 현재 사용자/구독관리 페이지에 하드코딩된 안내 문구를 관리 가능하게 할지 결정
- 완료 기준:
  - 정책 변경 시 코드 수정 최소화 여부가 결정됨
