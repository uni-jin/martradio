## 마트라디오 기능 개발 To-do 정리

### 1. 전체 플로우 개요
- **목표**: 마트 담당자가 서비스에 가입 → 체험 → 구독 결제 → 방송 생성/재생까지 자연스럽게 이어지는 사용자 여정 설계 및 구현.
- **대략적인 플로우**
  - 랜딩/홈 → 회원가입(추천인코드 포함) → 이메일 인증/로그인
  - 첫 로그인 시 온보딩/체험 플로우 진입
  - 체험(예: 무료 방송 1~N회 or N일 제한) 후 구독 플랜 선택
  - 구독 플랜에 따라 방송 글자수 제한 및 기능 제약 적용
  - 새 방송 만들기/기존 방송 열기 화면에서 방송 생성·재생

---

### 2. 회원 가입 및 로그인

- **요구사항 정리**
  - **회원 유형**: 마트 담당자(1차적으로만 고려, 롤은 `ROLE_USER` 정도로 단순 시작).
  - **입력 항목(회원가입 폼)**
    - 필수
      - 마트명
      - 사업자 등록번호(필요 여부는 옵션, 우선 텍스트 필드로 정의만)
      - 담당자 이름
      - 담당자 이메일(로그인 ID)
      - 비밀번호 / 비밀번호 확인
      - 연락처(휴대폰 번호)
    - 선택
      - 추천인 코드
  - **기능**
    - 추천인 코드 유효성 검증(존재하는 코드인지 확인, 없으면 경고만 노출하고 가입은 가능하게 할지 여부 결정 필요).
    - 이메일 인증 여부(우선은 옵션, 1단계에서는 미도입 가능. 이후 확장 고려).
    - 로그인/로그아웃, 비밀번호 재설정(이메일 링크 기반 or 추후 도입).

- **데이터 모델(초안)**
  - `users` 테이블
    - `id` (PK)
    - `email` (unique)
    - `password_hash`
    - `name` (담당자 이름)
    - `phone`
    - `mart_name`
    - `business_number` (nullable)
    - `referral_code_used` (nullable, FK-like to `referral_codes.code`)
    - `created_at`, `updated_at`, `last_login_at`
  - `referral_codes` 테이블
    - `id` (PK)
    - `code` (unique)
    - `owner_user_id` (누가 이 코드를 가진 추천인인지, 추후 관리자/파트너 계정과 연동)
    - `description` (nullable)
    - `is_active`
    - `created_at`, `updated_at`

- **프론트엔드 구현 To-do**
  - [ ] 회원가입 페이지 라우트 정의(예: `/signup`).
  - [ ] 회원가입 폼 UI 설계(마트 정보/담당자 정보/추천인코드 입력).
  - [ ] 패스워드 유효성 검증(길이/복잡도 규칙).
  - [ ] 추천인 코드 입력 필드 및 도움말 문구 추가.
  - [ ] 가입 성공 시 로그인 페이지 또는 자동 로그인+온보딩 진입 UX 정의.

- **백엔드/API To-do**
  - [ ] `POST /api/auth/signup` (회원 생성 및 추천인 코드 기록).
  - [ ] `POST /api/auth/login` (세션/토큰 발급).
  - [ ] 비밀번호 해시/검증 로직 구현.
  - [ ] 추천인 코드 유효성 체크 로직 및 통계 집계용 이벤트 기록(아래 5번과 연계).

### 3. 회원가입 후 체험 기능

- **요구사항 정리**
  - 새로 가입한 사용자가 **정식 결제 전**에 서비스가 어떤 것인지 체험.
  - 체험 형태 후보
    - **A안**: 방송 N회(예: 3회)까지 무료 생성/다운로드 가능.
    - **B안**: 가입 후 N일(예: 7일) 동안 무료 사용, 다만 글자수 제한은 가장 낮은 플랜 기준.
  - 최초 버전에서는 A안/B안 중 하나만 선택 구현.

- **데이터 모델/플래그**
  - `users` 테이블 또는 별도 `user_trial` 테이블
    - `trial_started_at`
    - `trial_expires_at`
    - `trial_available_generations` (체험 가능 방송 생성 횟수, 사용 시 감소)
    - `trial_status` (active/expired/consumed 등)

- **정책/로직 To-do**
  - [ ] 최초 회원가입 시 체험 정보 초기화 로직.
  - [ ] 방송 생성 시 체험 상태/잔여 횟수 또는 기한 체크.
  - [ ] 체험이 끝나면 구독 결제 유도 모달/배너 노출.

- **프론트엔드 To-do**
  - [ ] 첫 로그인 시 “간단 체험 안내” 온보딩 화면 구성.
  - [ ] 새 방송 만들기 화면 진입 시 체험 중/체험 종료 상태에 따른 안내 문구 처리.

### 4. 구독 결제 및 상품 플랜

- **요구사항 정리**
  - 기간 기반 플랜: **1개월 / 12개월**.
  - 마트 규모에 따른 플랜:
    - 소형마트: 200자 제한
    - 중형마트: 1000자 제한
    - 대형마트: 글자수 무제한
  - 추후 가격 정책/결제 PG 연동은 2단계로 진행(현재는 플랜/권한 구조만 설계).

- **데이터 모델(초안)**
  - `plans` 테이블
    - `id`
    - `name` (소형마트/중형마트/대형마트)
    - `description`
    - `duration_months` (1 또는 12)
    - `max_chars_per_broadcast` (null = 무제한)
    - `price` (금액, 추후 결정)
    - `created_at`, `updated_at`
  - `subscriptions` 테이블
    - `id`
    - `user_id`
    - `plan_id`
    - `started_at`
    - `expires_at`
    - `status` (active/cancelled/expired 등)

- **로직 To-do**
  - [ ] 사용자의 현재 활성 구독 및 플랜 조회 유틸리티 구현.
  - [ ] 새 방송 생성 시 글자수 제한 체크 로직 구현.
  - [ ] 체험 계정의 경우 구독보다 우선 적용될 정책 정의(예: 체험 중에는 별도 제한 사용 등).

- **프론트엔드 To-do**
  - [ ] 구독 플랜 선택 페이지(예: `/pricing` 또는 `/subscribe`) UI 설계.
  - [ ] 플랜별 글자수 제한 및 가격 설명.
  - [ ] 결제 버튼/결제 상태 표시(실제 결제 연동은 이후 단계에서 연동).

### 6. 새 방송 만들기 화면 개편

- **현재 요구사항 요약**
  - 기존: 기본 정보, 상품, 방송 멘트 입력 → 저장 → MP3 생성 화면으로 넘어가는 구조.
  - 변경: **한 화면에서** 제목 + 방송 내용 + (선택) 배경 음악 설정 후 바로 MP3 생성.
  - 행사 유형/시작/종료 등 복잡한 항목 제거.

- **UI/UX 설계 To-do**
  - [ ] 라우트: `/broadcast/new` 혹은 기존 경로 유지.
  - [ ] 입력 필드
    - 제목(필수)
    - 방송 내용(필수, 텍스트 영역, 글자수 카운터 표시 및 플랜/체험 제한 적용)

      - 유튜브 URL 입력 필드
      - 시작 시간(초 단위)
      - 종료 시간(초 단위)
      - “미리듣기” 버튼
  - [ ] MP3 생성 버튼
    - 조건: 필수 항목 유효성 통과 시 활성화.

    - 프론트에서 유튜브 player(JS SDK 등)를 활용하여 지정 구간 재생.

- **UI/UX To-do**

  - [ ] 테이블/카드형 리스트 구성
    - 제목
    - 생성 일시
    - 마지막 방송일시
    - (선택) 바로 재생 버튼 or 상세 보기 진입 버튼.
  - [ ] 정렬/검색
    - 기본 정렬: 최근 생성 or 최근 방송일시 기준.
